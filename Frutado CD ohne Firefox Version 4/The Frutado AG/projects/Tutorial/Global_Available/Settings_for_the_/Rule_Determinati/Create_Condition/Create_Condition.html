Create Condition Table

A condition table is a definition of characteristics or criteria that determine the rule usage.

In the rules-based availability check the condition tables hold condition records that define the specific access to a certain rule. During rules-based ATP check the condition records are read from the condition tables. The result depends on the fields in the field catalog that you choose when you create a new condition table. You determine the sequence of the key fields according to when you copied them from the field catalog. The selected fields determine the key for the condition records.

When the systems searches a condition table, the system finds a condition record that is consistent with the specific field values from the sales order that activates the ATP check.

Each successful access to a condition record returns a rule at runtime of the system.

You can define several condition tables in the system. Only specific ones are searched during the rules-based availability check. The access sequences are used to check this.
