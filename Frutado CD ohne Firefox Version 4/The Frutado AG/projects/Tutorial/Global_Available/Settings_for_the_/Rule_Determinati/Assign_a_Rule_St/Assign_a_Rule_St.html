    Assign a Rule Strategy

In this step you define the activation parameters used to determine a rule strategy and assign them to the previously defined rule strategy. Usually the values for the activation parameters come from a customer order that entered the OLTP-system. When you carry out the rules-based availability check the parameters are transferred to SAP APO. 

You have to define the following activation parameters:

  • Technical scenario: is consistent with the communication technology that activated rules-based availability check. The values are hard-coded in the ERP-system and are transferred to SAP APO from the calling application.
  • Business transaction: corresponds to the business context in which a rules-based availability check should take place. Usually the value is transferred to SAP APO from the calling OLTP-system at runtime and assigned to an order type.
  • Action type: corresponds to the manipulation technique of the calling transaction. These values are hard-coded in the ERP-system and are transferred to SAP APO from the calling application.

Note: Normally the values of the activation parameters have to comply with the values that are transferred from the connected OLTP-system. If this is not guaranteed, the system does not find any rule strategies or rules.
