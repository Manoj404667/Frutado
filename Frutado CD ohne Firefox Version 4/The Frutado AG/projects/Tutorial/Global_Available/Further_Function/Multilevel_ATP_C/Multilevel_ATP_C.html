Multilevel ATP Check

The multilevel ATP check is usually used in industry areas where a large part of the value-added activites occur at final assembly.The assemblies on the lower production levels have already been produced before the sales oder enters the system.

The sales order (mainly for configured products) can only be accepted when all components needed to produce the requested endproduct are available in time. However, the product configuration determined by the customer decides which components are needed specifically. This can only be checked at the time of sales oder entry.

In contrast to the CTP check initially no receipt elements are created during the multilevel ATP check. Instead, the checking results are stored in the so-called ATP tree structure. Receipts elements are generated later when the ATP tree structure  is transferred in the PP/DS module. This permits the system improved performance during the check. However, the conclusions on capacity availability and component availability can not be as detailed as in the CTP module.
