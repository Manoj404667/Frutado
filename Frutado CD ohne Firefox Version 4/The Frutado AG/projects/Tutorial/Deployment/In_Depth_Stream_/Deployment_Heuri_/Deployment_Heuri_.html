Deployment Heuristic Run

Running deployment heuristic creates a distribution plan by considering all available products at source locations as ATD quantity and then distributing them to cover demands at destination locations.

One important difference between deployment optimization and heuristic run is that the optimization method deploys products such that all supply sources are considered together to generate an optimal solution to the integrated supply chain, while the heuristic method considers each source separately thereby constituting a non-integrated distribution approach.

 

Furthermore, unlike deployment optimizer, deployment heuristic sticks to the sourcing decisions previously made in the SNP plan. Therefore, in order to run it, the system has to be initiliazed by running an SNP heuristic or optimizer to generate SNP stock transfers, or by running supply and demand propagation, or capable-to-match (CTM), to define the sourcing of the orders.   

 

Another difference is that deployment heuristic does not change any deployment order produced during the previous planning runs while deployment optimizer can change or delete the previously generated deployment orders to develop a plan anew.

 

In this lesson, we show how to run a deployment heuristic instance in supply network planning in the background.
