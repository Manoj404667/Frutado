'Cost Profile'

 

Defining a Cost Profile

A cost profile is defined to maintain the common cost terms as well as penalties applied when running a TP/VS optimization model under a planning scenario. In the TP/VS module, some of these costs are service related optimization measures which are hard to quantify. However, they should be defined such that an appropriate trade-off is obtained between different optimization measures of the TP/VS problem.

In the Frutado project, we define the following cost components in the cost profile.

  • Location-relevant costs including earliness, lateness, and non-delivery costs of an order at each location.
  • Transportation-relevant costs including fixed transportation costs per vehicle.
  • Dimension-relevant costs including variable transportation costs per certain units of measurement, e.g. duration and distance.
