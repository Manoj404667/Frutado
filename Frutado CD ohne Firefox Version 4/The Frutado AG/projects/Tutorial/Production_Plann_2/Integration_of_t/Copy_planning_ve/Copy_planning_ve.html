Copy the planning version

For the PP/DS we need to load daily demands into the liveCache. To prevent demands from occuring double, beforehand the liveCache needs to be purged from the aggregated demands that were used in the SNP. However, for SNP we still want to keep the data, so in a first step a new planning version FRU_PV_PP is created as a copy of planning version FRU_PV_SNP.
