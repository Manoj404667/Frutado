'Setup Activities'

Create Sequence-Dependent Setup Activities

In the Frutado AG, setup activities are required when changing the bottle type and the flavor of the beverage. These are considered to be sequence dependent. E.g., changing the flavor from a light flavor like Banana to a strong like Cherry takes less time than vice versa.

The sequence-dependent setup activities are represented by setup matrices. A setup matrix has to be created for each production resource. To each setup activity, a setup group or setup key is assigned to determine the required setup time and the associated costs.

The setup matrices indicate the possible setup transitions of the predecessor to the successor activity as well as the setup times and setup costs. While it would be possible to define all setup transitions using a nXn matrix (n being the number of products), this would require a big table which would cause stress to the performance of the system  because the system has to search for the associated setup transition whenever there is an order planned. The fact that usually many products have very similar setup requirements can be exploited to reduce the number of entries in the table, by splitting the definition of setups into setup groups and setup keys.

Setup groups can contain similar products, which have (mostly) the same setup transitions (in regard to time and costs). A so called standard setup transition indicates the setup transition between two setup groups. If exception setup transitions between certain products of the groups are different from the standard setup transition, individual exception setup transitions can be defined. These setup transitions are represented by setup keys which are assigned to the setup groups.

Remark: If setup activites are not sequence dependent, setup groups and matrices are not required. Then, the setup times and costs are maintained directly in the PP/DS-PPMs.
