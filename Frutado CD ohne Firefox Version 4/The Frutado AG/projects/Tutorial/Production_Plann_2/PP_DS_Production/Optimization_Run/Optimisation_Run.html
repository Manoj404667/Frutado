'Optimization Run'

Execute the PP/DS Optimization

You implement optimization in PP/DS after a heuristic run has been executed to improve the solution. 

You can use optimization to improve the production dates and the resource allocation for operations in regard to one ore more of the following criterias. The objective function is defined in the optimization profile:

  • Makespan
    • In SAP APO this is defined as the time span between the earliest possible start time of a production plan specified by the user and the end of the last  operation in the optimized plan.

  • Total Setup Times and Total Setup costs
    • You can use these criteria to take setup times and costs into account. The system determines setup times and setup costs that are dependent on sequence from the setup matrix.

  • Delay Costs
    • With these criteria you can improve the schedule according to the costs that are associated with the delayed completion of orders or activities. The following optimization criteria are available:

    • The sum of the delay costs
    • The maximum delay costs.
    There is a delay when:

    • A receipt date is later than a requirement date
    • The latest allowable start or finish date for an activity is before the actual start or finish date.

  • Mode costs
    • The mode costs for an activity are the costs that are incurred when executing an activity in a particular mode. You maintain the fixed and variable costs in the optimization profile depending on the mode priority.

The weightings that you apply to the various times or costs in the optimization profile when defining the objective function reflect the importance of each problem for your schedule.

For the optimization a Genetic Algorithm, which is a stochastic meta-heuristic, is employed. It works by modifying the existing schedule of the orders to find a better result according to the objective function. It may be set to end after a certain time or if no more improvements can be found. Note that in both cases the final result is not necessarily the true optimum. The sequence with the best objective value can then be saved or serve as a starting point for further interactive planning.
